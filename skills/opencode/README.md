OpenCode-specific repo skills would live here when the shared cross-client skill set is not sufficient.

At present, the managed OpenCode target is expected to use the shared skills under `skills/shared/` through the configured `skill-<name>` agent prefix, so this directory is intentionally a placeholder.

Each direct child directory with a `SKILL.md` would be rendered only when the OpenCode target exposes this directory through its own managed skill roots or agent-entry configuration.

Add repo-local OpenCode skill content here only when OpenCode needs behavior that should not be pushed into the shared cross-client skill set.
