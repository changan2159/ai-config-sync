# Browser Dashboard Validation Checklist

## 1. 单页基础刷新
- 打开首页，确认右上角显示 `本页自动刷新 · 每15s`
- 首次加载后看到 `刷新于 HH:MM:SS`
- 点击 `检测`
  - 页面数据立即刷新
  - 时间戳更新
  - 无报错横幅时保持隐藏

## 2. 页面隐藏暂停
- 打开单页后切到其他标签页或最小化窗口
- 返回前等待超过 15 秒
- 确认隐藏期间不会继续本页轮询
- 切回页面后应恢复刷新
- 右上角文案应反映状态切换：
  - 隐藏时：`当前页面已暂停`
  - 返回后：`本页自动刷新`

## 3. 多标签页共享一个自动刷新计时
- 同时打开 2 个可见标签页 A / B
- 确认只有一个标签页显示 `本页自动刷新`
- 另一个标签页显示 `由其他页面刷新`
- 等待一次自动刷新周期，确认两个页面数据都更新
- 关闭 leader 页或切到后台，确认另一个页面接管自动刷新

## 4. 安装/升级后的临时提频
- 触发任意安装/升级操作
- 任务完成后确认右上角变成 `每5s`
- 持续一段时间后自动恢复 `每15s`
- 版本号应在短时间内更新，不应长时间卡旧值

## 5. 守护启动/停止/重启后的临时提频
- 分别测试启动、停止、重启
- 操作后确认右上角变成 `每3s`
- 状态标签应快速收敛到最新结果
- 稳定后恢复 `每15s`

## 6. 错误展示
- 人为制造 `/api/status`、`/api/self`、`/api/processes` 的 403 或 500
- 确认会出现顶部红色 `刷新失败` 卡片
- 错误文案包含接口路径、HTTP 状态码、detail
- 点击 `立即重试` 后，在服务恢复时错误卡片消失
- 手动触发的设置、MCP、提示词、skill 删除等失败时应弹窗提示

## 7. MCP 操作错误
- 测试添加 MCP 的失败场景
- 测试删除 MCP 的失败场景
- 确认失败不会静默吞掉
- 成功时仍能正常打开任务日志并刷新列表

## 8. 设置错误
- 打开设置时模拟接口失败
- 保存 sudo 密码时模拟 403/400
- 清除 sudo 密码时模拟失败
- 保存 Web Host 时模拟失败
- 以上都应有明确错误提示

## 9. 提示词与 skill 操作错误
- 打开提示词弹窗时模拟读取失败
- 保存提示词时模拟失败
- 删除 skill 时模拟失败
- 确认弹窗或页面内能看到明确错误，不是静默无响应

## 10. 回归检查
- 进程计数仍能更新
- MCP 列表仍能正常显示
- Agent 列表仍能正常显示
- 打开 Agent 详情时自动刷新不应中断
- 控制台无新的语法错误
