- Dashboard agent cards should distinguish shared skills, target-specific skills, and effective managed skills. Showing only one total skill count is misleading for this repo because shared roots and per-target roots differ (`skills/shared`, `skills/codex`, `skills/pi`).
- Prompt management should expose three layers per tool: shared prompt source (`prompts/shared-global-prompt.md`), tool overlay prompt source (`prompts/*-global-prompt.md`), and effective generated prompt written into the client home (for example `~/.codex/AGENTS.md`). Source layers are editable; effective layer is read-only and regenerated after saves.
- Paseo is not a systemd service here; it is a local daemon managed via the `paseo` CLI. Dashboard wording should therefore use a local-daemon concept rather than systemd-service wording for Paseo.
- Tool cards should render Paseo first because it is the overall management daemon for this setup, and its card needs the most prominent placement in the runtime overview.
- MCP cards should include their own process lists and PID controls. For Serena, versioning/upgrades target `serena-agent`; `serena-manager` is repo-local launcher/runtime management and should be shown as manager context, not as the upgrade target.
- Pi's managed extensions should be shown as a dense version table with current version, latest version, per-row upgrade action, and one full-upgrade action. A tag cloud is not sufficient once custom components are being maintained individually.
- Raw process state `sleeping` from psutil commonly means an idle waiting daemon/session, not a stopped process. UI should translate it to a waiting/idle label while preserving the raw status for diagnostics.
- MCP process management should use the same grouped modal pattern for all managed MCPs, not only Serena. When an exact project path cannot be inferred, surface an explicit “未归属会话 · owner#pid” group instead of flattening all processes into one long list.
- Paseo card copy should not imply that Desktop and daemon versions are the same number. Show daemon/CLI/Desktop versions separately and add a short inline mismatch note when the Desktop client version differs from the running daemon version.
